2Deployment Instructions

For GitHub Pages:

  1. Create a GitHub account if you don't have one
  2. Create a new repository
  3. Upload your downloaded HTML file (rename it to index.html)
  4. Go to repository Settings â†’ Pages
  5. Select "main" branch and click Save
  6. Your site will be published at https://username.github.io/repository-name

For Netlify:

  1. Create a Netlify account at netlify.com
  2. Drag and drop your HTML file to the deployment area
  3. Netlify will provide you with a unique URL
  4. Optionally, connect to GitHub for continuous deployment

For Traditional Web Hosting:

  1. Purchase a web hosting plan and domain name
  2. Access your server via FTP or file manager
  3. Upload your HTML file to the public_html directory
  4. Rename the file to index.html if you want it as homepage
  5. Access your site via your domain name

How to Use Your WebChat

Learn how to effectively use your chat application.

1Accessing the Chat

Once deployed, users can access your chat by visiting the URL where you've hosted it.

No registration is required - users simply enter a username to start chatting.

2Sending Messages

To send a text message:

  1. Type your message in the input field at the bottom
  2. Press Enter or click the send button

To share media files:

  1. Click the appropriate media button (image, video, or file)
  2. Select the file from your device
  3. Add an optional text message
  4. Press Enter or click the send button

3Managing Contacts

The application includes a contact list on the left side:

  • See who's online
  • View message previews
  • Switch between different conversations

4Technical Notes

This is a frontend-only application:

  • Messages are not permanently stored
  • All users in the same session can see messages
  • For persistent messaging, consider adding a backend service
